# cloud/ ‚Äî FastAPI podcast server

Serves the RSS feed and streams MP3s from `OUTPUT_DIR` (a Google Drive folder
synced down to this machine). The local scraper writes there; this app reads
from there. Nothing is cached on disk ‚Äî the feed is built per request,
audio is streamed straight from the file with HTTP Range support.

## Setup

```bash
uv sync
cp .env.example .env
# edit .env: OUTPUT_DIR (the synced Drive path) and FEED_BASE_URL (this app's public URL)
```

## Run

```bash
uv run uvicorn app:app --host 0.0.0.0 --port 8000
```

Behind a reverse proxy (Caddy, nginx, Cloudflare Tunnel, Tailscale Serve/Funnel,
‚Ä¶), terminate TLS upstream and forward to `127.0.0.1:8000`. Make sure the
proxy passes through `Range`, `If-None-Match`, and `If-Modified-Since`
headers ‚Äî they all already work in this app.

## Endpoints

| Path | Purpose |
|---|---|
| `GET /` | One-line landing page pointing at `/feed.xml`. |
| `GET /feed.xml` | RSS 2.0 + iTunes + Podcasting 2.0 feed generated by scanning `OUTPUT_DIR`. |
| `GET /audio/{filename}` | Streams an `.mp3` from `OUTPUT_DIR`. Honours `Range:` headers (single-range only). Sets `ETag`, `Last-Modified`, `Cache-Control`. |
| `GET /transcripts/{filename}` | Serves a WebVTT (`.vtt`) transcript from `OUTPUT_DIR`. Linked from each item via `<podcast:transcript>`. |
| `GET /images/{filename}` | Serves a `.png` cover image from `OUTPUT_DIR`. Used for both per-episode `<itunes:image>` and the show-level `FEED_IMAGE_URL`. |

`FEED_BASE_URL` must point at this app ‚Äî enclosure URLs are constructed as
`${FEED_BASE_URL}/audio/<filename>.mp3`, transcript URLs as
`${FEED_BASE_URL}/transcripts/<filename>.vtt`, image URLs as
`${FEED_BASE_URL}/images/<filename>.png`. When a `.vtt` sibling exists
next to an `.mp3`, the item automatically grows a
`<podcast:transcript type="text/vtt" lang="‚Ä¶">` tag (the language is the
first segment of `FEED_LANGUAGE` ‚Äî `en-us` ‚Üí `en`). When a `.png` sibling
exists (written by [local/coverart.py](../local/coverart.py)), the item
also grows an `<itunes:image href="‚Ä¶">` tag.

## Show-level cover art

Drop a `cover.png` into `OUTPUT_DIR` and set:

```
FEED_IMAGE_URL=${FEED_BASE_URL}/images/cover.png
```

Apple recommends ‚â•1400√ó1400 PNG/JPG. Per-episode covers from `coverart.py`
default to 512√ó512; bump `COVER_WIDTH`/`COVER_HEIGHT`/`COVER_STEPS` in
[local/.env](../local/.env) if Apple Podcasts ignores them.

## Bare-MP3 mode

The feed iterates every `*.mp3` in `OUTPUT_DIR`. If a sibling
`<basename>.json` (written by the scraper) exists, its metadata is used;
otherwise a minimal record is synthesised from the filename + mtime, so you
can manually drop any audio file into the folder and it'll appear in the
feed with a stable, filename-derived GUID.
